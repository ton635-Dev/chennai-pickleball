# ピックルボール コミュニティ管理 Webアプリ 仕様書

**バージョン:** v1.0
**作成日:** 2026-07-19
**ステータス:** 仕様・デザイン方針とも確定。実装開始

---

## 1. 概要

### 1.1 背景と目的

インド・チェンナイで活動するピックルボールサークルの運営を支援するWebアプリ。
参加人数の増加に伴い、活動日の調整・参加者管理・試合運営などをアプリで一元化する。

### 1.2 コンセプト

- **みんなで使える:** 認証不要、URLを知っていれば誰でも名前を登録してすぐ使える
- **コート際で使える:** スマホ片手・屋外・日差しの下でも操作できるモバイルファーストUI
- **運営をラクに:** 活動日の出欠取りまとめ、大会の組み合わせ作成を自動化

### 1.3 想定ユーザーと規模

| 項目 | 内容 |
|---|---|
| 利用者 | サークルメンバー 20〜50人程度 |
| UI言語 | 日本語のみ |
| 主な利用端末 | スマートフォン(コート際での利用が中心)、一部PC |
| 同時利用 | 活動日に10〜20人程度がアクセスする想定 |

### 1.4 開発・公開の流れ

1. チャットで仕様書を作成・ブラッシュアップ(← いまここ)
2. 仕様が固まったら Claude Code で実装
3. ある程度実装が完了したら Vercel に公開
4. コアメンバーで試用 → 不足機能・改善点を洗い出し → 修正のサイクル

## 2. 技術構成

| レイヤ | 技術 | 備考 |
|---|---|---|
| フロントエンド | Next.js(App Router)+ TypeScript | React Server Components 活用 |
| UI | Tailwind CSS + shadcn/ui | モバイルファースト。スコアボードは専用大型UI |
| バックエンド | Supabase(PostgreSQL) | Next.js の Server Actions / Route Handlers 経由でアクセス |
| リアルタイム | Supabase Realtime | スコアボードの観戦表示・出欠のライブ更新に使用 |
| ホスティング | Vercel(無料枠) | Supabase も無料枠で開始 |
| PWA | next-pwa 等でホーム画面追加に対応 | オフライン対応はスコアボードのみ検討【要確認】 |

**無料枠での運用について:** 20〜50人規模なら Vercel / Supabase ともに無料枠で十分。
Supabase 無料枠は1週間アクセスがないと一時停止するため、定期ping(cron)を仕込むか、週1回以上の利用があれば問題なし。

## 3. ユーザーモデル(認証なし方式)

### 3.1 基本方針

- ログイン・パスワードは**なし**
- 初回アクセス時に「名前(表示名)」を登録するとメンバーとして作成される
- 端末のブラウザにメンバーIDを保存(localStorage)し、次回以降は自動で本人として扱う
- 機種変更・別端末からは「メンバー一覧から自分を選ぶ」ことで復帰できる

### 3.2 なりすまし・いたずら対策(小規模コミュニティ前提の緩い対策)

- アクセス制限は設けない。**アプリのURLを非公開にしてメンバーにのみ共有する運用**とする【確定】
- 操作履歴(誰がいつ変更したか)を主要データに記録し、トラブル時に追跡できるようにする

### 3.3 権限(ロール)— 全員フラット【確定】

- ロールの区別は設けず、**全メンバーがすべての操作を実行可能**(活動日の作成・編集、大会の作成・管理を含む)
- 誤操作・トラブル対策として以下を実装する:
  - 主要データの変更に**操作履歴(誰がいつ何を変更したか)**を記録
  - 削除系の操作は確認ダイアログ必須。活動日・大会の削除は「アーカイブ(論理削除)」とし、復元可能にする

## 4. 機能仕様

### 4.1 ルール学習機能(初心者向け)

**目的:** 新規参加者が活動前・活動の合間にルールを自習できる。

#### コンテンツ構成(章立て)

1. **ピックルボールとは** — 競技の概要、コート・道具の紹介
2. **コートとポジション** — コート図解(キッチン=ノンボレーゾーンの説明)
3. **サーブのルール** — アンダーハンド、対角線、ワンバウンド等
4. **ツーバウンドルール** — サーブ後の2バウンドの図解
5. **ノンボレーゾーン(キッチン)** — 反則になるケースの図解
6. **スコアリング** — 点の数え方、コール方法(0-0-2 など)
7. **よくある反則・Q&A**

#### 機能

- 章ごとのページ + SVG/イラストによる図解(コート図はアプリ内で描画)
- 各章末に**理解度クイズ**(3択、5問程度)。全問正解でバッジ表示
- 「試合中にすぐ見たい」向けの**早見表ページ**(1画面でルール要点)
- 静的コンテンツとして実装(DB不要、Markdown/MDX管理でメンテしやすく)

### 4.2 スコアリング機能(試合用スコアボード)

**目的:** 試合中に誰でも簡単にスコアを付けられ、観戦者にも見やすく表示する。

#### 前提ルール【確定: サイドアウト式のみ・点数は自由入力】

- サイドアウト・スコアリング:**サーブ権を持つ側のみ得点**(伝統的な公式ルール)
- ゲーム点数は**自由入力**(デフォルト11点)・2点差で終了(デュース時は続行)
  - 練習・時間調整用に7点や15点など任意の点数を試合開始時に設定できる
- ダブルスはサーバー番号あり(スコアコールは「自分-相手-サーバー番号」、開始は 0-0-2)
- シングルス / ダブルス両対応
- ラリーポイント式は実装しない(将来必要になれば拡張できる設計にはしておく)

#### UI仕様

- **超シンプル操作:** 画面を左右に2分割し、得点した側の領域をタップするだけ
  - サーブ権の移動・サーバー番号・サイドアウトはアプリが自動判定
- 大きなスコア表示(屋外でも見やすい高コントラスト・大フォント)
- 現在のサーバーとサーブ位置(右/左)をコート図でハイライト表示
- **元に戻す(Undo)** ボタン — タップミス対策として必須
- ゲーム終了(設定点数到達・2点差)を自動判定し勝敗を記録
- 横画面表示にも対応(タブレットをベンチに置いて観戦用スコアボードに)

#### データ・連携

- 試合結果(対戦者、スコア、日時)を保存し、活動日・大会に紐づけ可能
- カジュアルモード(記録を残さないその場だけのスコアボード)も用意
- Supabase Realtime でスコアを配信し、別端末から観戦できる(コート横の1台で入力 → みんなのスマホで閲覧)

### 4.3 コート情報記録機能

**目的:** チェンナイ市内で使用するコートの特徴・ノウハウをコミュニティで蓄積する。

#### 記録項目

| 項目 | 内容 |
|---|---|
| コート名 | 施設名・通称 |
| 場所 | 住所 + Google Maps リンク |
| 屋内/屋外 | 屋外の場合は風・日差しの傾向メモ |
| 面数 | 使用可能なコート数 |
| 路面 | ハード/人工芝/体育館床 等、バウンドの特徴 |
| 設備 | ネット常設有無、照明(ナイター可否)、更衣室、駐車場、飲料 |
| 料金 | 時間あたり料金、支払い方法 |
| 予約方法 | 電話/アプリ/現地 等、予約のコツ |
| 写真 | 複数枚アップロード(Supabase Storage) |
| メモ | 自由記述(雨天時の状態、混雑時間帯など) |

#### 機能

- コート一覧(カード表示)・詳細ページ
- メンバーなら誰でも追加・編集可能(編集履歴を記録)
- 活動日作成時にコートを選択 → 活動詳細に場所情報が自動表示

#### 項目別評価 + コメント【確定】

メンバー各自が、コートを**項目ごとに5段階評価 + コメント**できる。

| 評価項目 | 観点の例 |
|---|---|
| コート数 | 面数は十分か、待ち時間が発生しにくいか |
| 路面(滑りやすさ) | グリップ、雨後・砂での滑りやすさ、バウンドの素直さ |
| 広さ | コート周囲のスペース、天井高(屋内)、隣コートとの間隔 |
| 設備 | ネット・照明・更衣室・駐車場などの充実度 |
| アクセス | 行きやすさ、予約の取りやすさ |

- コート詳細ページに項目ごとの平均点(★表示)とコメント一覧を表示
- 自分の評価はいつでも変更可能(1人1コートにつき1評価)
- 評価項目は運用しながら追加・変更できる設計にする(項目マスタとして持つ)

### 4.4 スケジュール・参加者管理機能

**目的:** 活動日の告知と出欠の取りまとめを一元化する。

#### 活動日(イベント)の作成 — 管理者

- 日付・開始/終了時刻
- コート(4.3のコートマスタから選択)
- 参加費(任意)
- 出欠締切(任意)
- 定員の設定は**なし**【確定】(人数調整は参加者リストを見ながら現場・WhatsAppで行う運用)
- メモ(持ち物、初心者歓迎、など)
- 繰り返し設定は当面実装しない(不定期・都度作成の運用を想定。定例化したら将来追加)

#### 出欠登録 — メンバー

- **参加 / 不参加 / 未定** の3択をワンタップで表明・変更
- コメント添付可(「遅れて行きます」など)
- 参加者リストをリアルタイム表示(参加◯名)

#### 表示

- ホーム画面に「次回の活動日」を大きく表示 + 自分の出欠状態
- 月間カレンダー表示 / リスト表示の切り替え
- 過去の活動日は参加実績として保存(誰が何回参加したかの集計に利用)

#### WhatsApp連携(告知サポート)【確定】

- 活動日を作成すると、**WhatsAppにそのまま貼れる告知テキストを自動生成**
  (日時・場所・Google Mapsリンク・出欠登録用URL を整形)
- 「WhatsAppで共有」ボタン(`wa.me` の共有リンク)でワンタップ共有
- 出欠の途中経過も「現在の参加者リスト」をテキストコピーしてWhatsAppに貼れる

### 4.5 大会管理機能(トーナメント / リーグ戦)

**目的:** コミュニティ内大会の組み合わせ作成・進捗管理・結果表示を自動化する。

#### 大会の作成 — 管理者

- 大会名、開催日(活動日と紐づけ)
- 形式: **トーナメント(シングルエリミネーション)** / **リーグ戦(総当たり)** の2種から選択
  - 発展形(ダブルエリミネーション、予選リーグ+決勝トーナメント、スイス式)は将来対応【要確認】
- 種目: シングルス / ダブルス(固定ペア) / ダブルス(ペアをランダム組成)
- 参加者登録: 活動日の参加者から選択、またはその場で追加

#### 進行管理

- 組み合わせ自動生成
  - トーナメント: シード・BYE(不戦勝)の自動配置
  - リーグ戦: 総当たり表の自動生成、コート数に応じた試合順の提案
- 各試合はスコアボード(4.2)と連動 — スコアボードで確定した結果が自動反映
- 結果の手入力にも対応(紙で進めた場合の後入力)

#### 表示

- **トーナメント表(ブラケット図)** — 勝ち上がりをリアルタイム表示、スマホでは横スクロール
- **リーグ戦の星取表 + 順位表** — 勝敗数・得失点差で自動順位付け
- 「次の試合」「コート割り当て」の表示で当日の運営をサポート
- 大会結果の履歴ページ(過去大会の優勝者一覧など)

## 5. 画面構成(サイトマップ)

```

    ├─ メンバー一覧(参加回数など)
    └─ プロフィール(表示名変更、端末の再ひも付け)
```


※ 実装時に Claude Code で詳細化(型・制約・インデックス・RLSポリシー)する。

## 7. 非機能要件

- **モバイルファースト:** 全画面がスマホ縦持ちで完結。スコアボードのみ横持ち対応
- **表示速度:** コート際の弱い電波でも使えるよう、初期ロードを軽量に
- **リアルタイム性:** 出欠・スコアは数秒以内に他端末へ反映
- **データ保全:** Supabase の自動バックアップ(無料枠の範囲)
- **コスト:** 月額0円運用を基本方針とする

## 10. デザイン指針【確定】

UIモックで方向性を確認済み(Claude Designプロジェクト「ピックルボール Chennai — UIモック」/ 実装リポジトリには `design-mocks/` として同梱)。実装時は各モックHTMLを見た目の基準とする。

### 10.1 カラートークン

| トークン | 値 | 用途 |
|---|---|---|
| Primary | `#0E7C63` | コートグリーン。主要アクション・勝ち上がり線 |
| Primary Dark | `#0A5C4A` | グラデーション・強調テキスト |
| Accent | `#D8EC3F` | ボールライム。CTA・スコア数字・選択状態 |
| Navy | `#122B33` | スコアボード背景・ダークUI |
| Ink | `#14211C` | 本文テキスト |
| BG | `#F4F7F5` | 画面背景 |
| Amber | `#E8A93C` | 未定ステータス・注意 |
| Gray | `#9AA8A2` | 不参加・非活性 |

### 10.2 コンポーネント基調

- カード: 白背景・角丸16px・薄いボーダー(#E2EAE5)
- ボタン: ピル型(角丸999px)・太字。WhatsApp共有はアクセント色
- 出欠は「参加/未定/不参加」の3色チップで統一
- フォント: システムフォント(Hiragino / Noto Sans JP)

### 10.3 レイアウト

- **スマホ(基準):** 下部タブナビ5項目(ホーム/予定/スコア/大会/その他)。390px幅基準
- **PC:** 上部水平ナビ+2カラムグリッド(最大幅 約1180px)
- **スコアボード:** ダークネイビー背景+ライムの大型数字。画面を上下2分割し、得点側をタップで加点。サーブ側をグリーン面+「サーブ#n」チップで表示。スコアコール(例 7-5-2)を上部に常時表示
- **トーナメント表:** FINALを頂点とする縦型ブラケット(決勝が上・準々決勝が下)。勝ち上がり確定=グリーン実線、対戦待ち=グレー破線。ラウンド進行ステッパー+「次の試合」カードを上部に配置。8チーム(4試合)規模まで横スクロールなしの1画面表示。16チーム超は折りたたみ表示を将来検討
